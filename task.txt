Результатом будет небольшое веб-приложение. 
Интерфейс - по желанию, главное - возможность стучаться по http и получать ответ хотябы в json.
В качестве ограничения - не пользоваться орм-ками, доступ только через отправку в бд строк sql-кода.
Можно пользоваться например стандартной либой sqlite3
Приложение можно разбить на две части - собственно сервер, и скрипт, который сетапит таблицы в бд.

таблица: колонки
user: id_, name, surname, age
pay: id_, date_, caption, user, thing, sum_
thing: id_, date_, name, additional_info, user, thing_type, sum_
thing_type: id_, name

id_ age sum_ типа int
колонки с названием другой таблицы - int и являются связью с этой таблицей (foreign key любого типа)
остальное - текст

Методы должны уметь crud над каждой таблицей.
Товар обязательно должен иметь пользователя, и может иметь оплат не более, чем он стоит.
Оплачивать товар может любой пользователь, не только владелеац.
Оплаты можно создавать только на существующий товар.
